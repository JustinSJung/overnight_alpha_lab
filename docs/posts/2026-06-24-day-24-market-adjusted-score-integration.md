# Day 24: Market-Adjusted Score Integration

## What I Built Today

Today, I added a market-adjusted score integration layer to Overnight Alpha Lab.

On Day 23, the system learned how to evaluate prediction results using market-adjusted returns.

Today, I converted those market-adjusted evaluation results into recommendation score adjustment signals.

This means the system can now reward predictions that outperform the market and penalize cases where the stock only appeared successful because the broader market moved in the same direction.

## Why This Matters

A stock can rise for many reasons.

It may rise because:

```text
the disclosure event was meaningful
the company had strong news
the whole market moved up
the sector moved up
```

Without market adjustment, the system may overestimate weak signals.

For example:

```text
Stock next-day return: +2.00%
Market next-day return: +3.00%
Market-adjusted return: -1.00%
```

In this case, the stock increased in absolute terms, but underperformed the market.

This should not receive the same score as a true event-driven success.

## New File

I added a new score integration module.

```text
src/models/market_adjusted_score_integrator.py
```

## Input Data

The score integrator uses the market-adjusted evaluation file.

```text
data/predictions/market_adjusted_evaluation_YYYYMMDD.csv
```

This file was generated by the Day 23 evaluator.

## Output Files

The score integrator generates:

```text
data/processed/market_adjusted_score_adjustments_YYYYMMDD.csv
reports/daily_review/YYYY-MM-DD_market_adjusted_score_report.md
```

The processed CSV may be ignored by Git depending on `.gitignore`, but the Markdown report is committed to the portfolio repository.

## Score Rules

The new score adjustment rules are:

```text
market_adjusted_success: +15
market_driven_weak_success: -5
relative_success_but_absolute_loss: +5
market_adjusted_failure: -15
relative_failure_despite_absolute_gain: -10
market_adjusted_volatility_success: +10
market_driven_volatility: -5
volatility_overestimated: -10
market_data_missing: 0
pending: 0
unknown: 0
```

## What This Adds

The system now creates:

```text
market_adjusted_score_adjustment
market_adjusted_adjustment_label
market_adjusted_adjustment_reason
```

These fields turn market-adjusted evaluation into a score signal that can later be connected directly to the daily stock recommender.

## Adjustment Labels

The score integrator also creates adjustment labels:

```text
strong_positive_adjustment
mild_positive_adjustment
neutral_adjustment
mild_negative_adjustment
strong_negative_adjustment
```

This makes the score impact easier to interpret in reports.

## Why This Is Progress

This step moves the project from evaluation to scoring.

Previously, market-adjusted evaluation was only a review layer.

Now, the evaluation result becomes a structured score adjustment signal.

This is important because the recommender can eventually use this signal to improve candidate ranking.

The system is gradually becoming less dependent on simple rule-based event scoring and more dependent on historical performance, market context, and post-event learning.
