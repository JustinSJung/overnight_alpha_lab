# Day 5: Baseline Machine Learning Model

## What I Built Today

Today, I added the first baseline machine learning model to Overnight Alpha Lab.

The model reads the machine-learning dataset generated by the pipeline and prepares it for supervised learning.

The current target is simple:

* `success` = correct event reaction prediction
* `failure` = incorrect event reaction prediction
* `pending` = excluded from training

This step turns the project from a data pipeline into a model-training framework.

## Why This Matters

A prediction system should not only collect data.

It should also be able to:

* evaluate whether past predictions were correct
* exclude data that cannot be judged yet
* train only on confirmed outcomes
* generate model reports
* improve as more daily data accumulates

The baseline model is the first version of that learning structure.

## Current Model Design

The first model uses a simple logistic regression classifier.

The goal is not to create a perfect model immediately.

The goal is to create a measurable baseline that can be improved over time.

## Input Features

The current feature set includes:

| Feature                | Meaning                                             |
| ---------------------- | --------------------------------------------------- |
| event_score            | Rule-based event score                              |
| news_count             | Number of related news items                        |
| positive_keyword_count | Count of positive keywords in news                  |
| negative_keyword_count | Count of negative keywords in news                  |
| news_sentiment_score   | Positive keyword count minus negative keyword count |
| news_attention_score   | Basic attention score based on news count           |
| event_type             | Disclosure event type                               |
| prediction_direction   | Initial expected direction                          |
| initial_confidence     | Initial confidence level                            |

## Target Variable

The target variable is:

```text
prediction_result
```

The model only uses rows labeled as:

```text
success
failure
```

Rows labeled as:

```text
pending
```

are excluded because next trading day price data is not available yet.

## Current Result

At this stage, the model ran successfully, but there were not enough trainable samples.

Current dataset status:

```text
Total dataset rows: 54
Trainable rows: 0
```

This is expected.

Most current rows are still pending because the next trading day has not arrived yet.

## Why This Is Still Progress

This result is important because the system is honest about data readiness.

Instead of forcing a model to train on incomplete data, the pipeline now checks whether valid training samples exist.

If there are not enough samples, it creates a report explaining why the model cannot train yet.

This is better than producing misleading accuracy numbers.

## Model Report

The baseline model also generates a daily model report.

The report explains:

* total dataset rows
* trainable rows
* whether the model was trained
* why training was skipped if data is insufficient
* what needs to happen next

This makes the system easier to review and improve over time.

## Next Step

The next step is to continue running the daily pipeline after new price data becomes available.

Once pending rows are converted into success or failure cases, the model will be able to train.

Future improvements may include:

* model evaluation metrics
* feature importance
* probability output
* confidence scoring
* LightGBM or XGBoost models
* automatic daily model comparison
* improved error-note feedback loop

The baseline model is only the starting point.

The real value will come from repeated daily simulation, review, and improvement.
